# Demo 與報告

## 目標

把作品呈現成「固定站牌離站決策 + 台語數位站務員」，避免 demo 被看成一般聊天機器人或弱化版 Google Maps。

## 狀態

Not started。需要依新版主軸重寫 demo flow 與報告素材。

## Demo 主流程

- [ ] 站牌狀態首頁：目前可搭、未發車、末班已過
- [ ] 數位站務員播報首頁摘要
- [ ] 單一路線查詢與播報
- [ ] 資料錯誤或 ASR 失敗 fallback
- [ ] 路線規劃作為 secondary 展示，不喧賓奪主

## 交付物

- [ ] Demo 腳本 10 分鐘版

- [ ] 時序圖
