# 評估與使用者試用

## 目標

用可量化與可觀察的方式證明本系統不是單純公車 chatbot，而是能降低固定站牌離站決策負擔。

## 狀態

Not started。產品方向已確定，評估設計與試用尚未執行。

## 指標

- [ ] 離站決策理解率：能否判斷現在可搭 / 末班已過 / 是否值得等
- [ ] 任務完成時間：傳統到站列表 vs 離站決策 dashboard
- [ ] 主觀負擔：NASA-TLX 或簡化 Likert 題
- [ ] 數位站務員幫助度：有 / 無台語播報比較
- [ ] First-audio latency：按「唸給我聽」後 p50 / p95
- [ ] ASR 短指令辨識率
- [ ] TTS 自然度 MOS 評分

## 使用者試用

- [ ] 訪談腳本
- [ ] 同意書
- [ ] 學生試用，N >= 10
- [ ] 長輩試用，N >= 5
- [ ] SUS 問卷整理
- [ ] 失敗案例分類：資料錯誤、理解錯誤、操作錯誤、語音錯誤
